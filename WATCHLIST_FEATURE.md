# 我的关注功能设计文档

## 📋 功能概述

实现用户自选股票关注功能，允许用户注册账号并管理最多5只自选股票。

## ✨ 核心特性

### 1. 用户系统
- **简单注册**：用户名 + 密码（不验证邮箱）
- **登录/登出**：基于session的简单认证
- **无邮箱验证**：降低注册门槛

### 2. 关注列表管理
- **添加股票**：搜索并添加股票到关注列表
- **删除股票**：从关注列表移除股票
- **数量限制**：最多关注5只股票
- **实时更新**：添加/删除后立即刷新列表

### 3. 智能缓存共享
- **共享机制**：多个用户关注同一只股票时，共享同一个缓存
- **节省成本**：避免重复的AI分析调用
- **一致性**：所有用户看到的分析数据一致

## 🗄️ 数据库设计

### users 表
```sql
- id: UUID (主键)
- username: VARCHAR(50) (唯一)
- password_hash: VARCHAR(255)
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
```

### user_watchlist 表
```sql
- id: UUID (主键)
- user_id: UUID (外键 -> users.id)
- stock_id: INTEGER (外键 -> stocks.id)
- symbol: VARCHAR(20) (冗余存储)
- added_at: TIMESTAMP
- UNIQUE(user_id, stock_id)
```

## 🔐 认证方案

### 简化的Session认证
1. 用户注册时，密码使用bcrypt加密
2. 登录成功后，在localStorage存储用户信息
3. 前端每次请求携带用户ID
4. 后端简单验证（不使用JWT，简化实现）

**注意**：这是简化实现，生产环境应使用更安全的方案（JWT、httpOnly cookies等）

## 🔄 API端点

### 认证相关
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/logout` - 用户登出

### 关注列表相关
- `GET /api/watchlist?userId=xxx` - 获取用户关注列表
- `POST /api/watchlist` - 添加股票到关注列表
- `DELETE /api/watchlist/:id` - 删除关注的股票
- `GET /api/stocks/search?q=xxx` - 搜索股票

## 🎯 缓存共享机制

### 工作流程
1. 用户A关注股票AAPL
2. 系统检查`stock_analyses`表是否有AAPL的当天缓存
3. 如果有缓存，直接使用
4. 如果没有缓存，调用AI分析并保存
5. 用户B也关注AAPL时，直接使用步骤4保存的缓存

### 优势
- **成本节省**：避免重复的AI调用
- **响应速度**：缓存命中时响应快
- **数据一致**：所有用户看到相同的分析
